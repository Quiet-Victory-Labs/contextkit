title: context rules
description: List all lint rules with tier, severity, and fixability metadata.
---

The `rules` command lists every lint rule RunContext ships, with metadata about which tier it belongs to, its severity, and whether it can be auto-fixed.

## Usage

```bash
context rules
```

## Options

| Option | Description | Default |
|---|---|---|
| `--tier <tier>` | Filter by tier: `bronze`, `silver`, `gold` | All tiers |
| `--fixable` | Show only auto-fixable rules | All rules |
| `--format <type>` | Output format: `pretty` or `json` | `pretty` |

## Examples

### List all rules

```bash
context rules
```

### Filter to Gold-tier rules only

```bash
context rules --tier gold
```

### Show only fixable rules

```bash
context rules --fixable
```

### JSON output for tooling

```bash
context rules --format json
```

## Rule categories

| Tier | Count | Focus |
|---|---|---|
| **Bronze** | 12 | Descriptions, ownership, grain, security, valid schema |
| **Silver** | 3 | Trust status, refresh cadence, upstream lineage |
| **Gold** | 14 | Semantic roles, aggregation rules, golden queries, relationships |
| **Data** | 9 | Source tables exist, fields exist, types compatible, queries execute |
