The ContextKit MCP server can be configured for any MCP-compatible client. You can run it **locally** (stdio or self-hosted HTTP) or connect to the **hosted endpoint** at `mcp.runcontext.dev`.

## Connection modes

| Mode | Transport | Use case |
|------|-----------|----------|
| **Local stdio** | stdin/stdout | Single-user, local development |
| **Local HTTP** | Streamable HTTP | Self-hosted, shared team server |
| **Hosted** | Streamable HTTP | Managed cloud, no server to run |

For the hosted endpoint, replace `{org}` with your organization slug and set `CONTEXTKIT_API_KEY` to your API key (available from the ContextKit dashboard).

## Claude Code

<Tabs>
<TabItem label="Local (stdio)">
```json title=".claude/mcp.json"
{
  "mcpServers": {
    "contextkit": {
      "command": "npx",
      "args": ["contextkit", "mcp"],
      "cwd": "/path/to/your/project"
    }
  }
}
```
</TabItem>
<TabItem label="Local (HTTP)">
```json title=".claude/mcp.json"
{
  "mcpServers": {
    "contextkit": {
      "url": "http://localhost:3000/mcp"
    }
  }
}
```
</TabItem>
<TabItem label="Hosted">
```json title=".claude/mcp.json"
{
  "mcpServers": {
    "contextkit": {
      "url": "https://mcp.runcontext.dev/{org}",
      "headers": {
        "Authorization": "Bearer ${CONTEXTKIT_API_KEY}"
      }
    }
  }
}
```
</TabItem>
</Tabs>

<Aside type="tip">
Set `cwd` to the directory containing your `contextkit.config.yaml` file. If the config is in the project root, you can omit `cwd`.
</Aside>

## Generic MCP client

Any MCP-compatible client can connect using either transport.

### Local (stdio)

Run the MCP server as a subprocess:

```bash
npx contextkit mcp
```

The server communicates over stdin/stdout using the MCP JSON-RPC protocol.

### Local (HTTP)

Start the server in HTTP mode:

```bash
npx @runcontext/cli serve --http --port 3000
```

Then point your client at:

```
http://localhost:3000/mcp
```

### Hosted

Send MCP JSON-RPC requests to the hosted endpoint:

```
POST https://mcp.runcontext.dev/{org}
Authorization: Bearer <your-api-key>
Content-Type: application/json
```

The hosted endpoint uses Streamable HTTP transport. Example `initialize` request:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-03-26",
    "capabilities": {},
    "clientInfo": {
      "name": "my-client",
      "version": "1.0.0"
    }
  }
}
```

## Verifying the connection

Once configured, your AI tool should discover the ContextKit resources and tools automatically. You can verify by asking the agent:

> "What MCP resources are available?"

It should list the `context://` resources. You can also test a tool call:

> "Search my semantic layer for revenue metrics."

The agent should call `context_search` and return results from your context directory.

## CLI flags

The `serve` command accepts the following flags:

| Flag | Description | Default |
|------|-------------|---------|
| `--http` | Use HTTP transport (default is stdio) | Off |
| `--port <number>` | Port for HTTP server | `3000` |
| `--host <address>` | Host address to bind | `0.0.0.0` |
| `--context-dir <path>` | Path to context files | `./context` |
