The Problem

AI agents don't understand
your data — yet

What catalogs provide

  • Table names and column types
  • Basic descriptions
  • "What exists" — not "how to use it"
  • No query patterns or rules

What AI agents need

  • Semantic roles (metric vs dimension)
  • Default aggregations & additivity
  • Guardrail filters & business rules
  • Golden query examples
The Consequence

Without context, AI generates wrong SQL

AI without context

-- "How many customers?"
SELECT COUNT(*)
FROM transactions

-- Wrong: counts transactions,
-- not distinct customers.
-- Also includes refunds.

AI with ContextKit

-- "How many customers?"
SELECT COUNT(DISTINCT customer_id)
FROM transactions
WHERE amount > 0

-- Correct: business rules,
-- guardrails, semantic roles.

Foundation

Built on Open Semantic Interchange v1.0

A vendor-neutral semantic model standard backed by Snowflake, dbt Labs, Salesforce, Alation, Sigma, ThoughtSpot, and others. ContextKit reads and validates OSI files natively — never modifies them.

OSI Defines

  • Datasets, fields, and expressions
  • Relationships and join paths
  • Metrics with SQL expressions
  • AI context (synonyms, instructions)

ContextKit Adds

  • Governance (ownership, trust, security)
  • Business rules & guardrails
  • Golden query examples
  • Lineage, glossary, tier scoring
Architecture

Everything is versioned YAML

context/
  models/ # OSI v1.0 semantic models
    retail-sales.osi.yaml
  governance/ # Ownership, trust, security
    retail-sales.governance.yaml
  rules/ # Business rules & queries
    retail-sales.rules.yaml
  lineage/ # Data flow declarations
    retail-sales.lineage.yaml
  glossary/ # Business term definitions
    revenue.term.yaml
  owners/ # Team/person registry
    analytics-team.owner.yaml
contextkit.config.yaml

Separation of Concerns

  • OSI files — pure semantic models, interoperable with any OSI tool
  • Governance — layers ownership, trust, security, field semantics
  • Rules — golden queries, business rules, guardrails, hierarchies
  • Lineage — upstream sources and downstream consumers
  • Glossary & Owners — reusable term definitions and team registry

All files are version-controlled, reviewable, and CI/CD-friendly.

Developer Experience

10 CLI commands. Zero config to start.

$ cd my-project

# Core workflow

Example

From YAML to AI-ready in minutes

# governance/retail-sales.governance.yaml
model: retail-sales
owner: analytics-team
trust: endorsed
security: internal

fields:
  transactions.amount:
    semantic_role: metric
    default_aggregation: SUM
    additive: true
    default_filter: "amount > 0"
  customers.region:
    semantic_role: dimension
    sample_values: [NA, EMEA, APAC]
# rules/retail-sales.rules.yaml
model: retail-sales

golden_queries:
  - question: "Revenue by region?"
    sql: |
      SELECT c.region,
        SUM(t.amount) AS revenue
      FROM transactions t
      JOIN customers c ...

guardrail_filters:
  - filter: "amount > 0"
    reason: "Exclude refunds"
