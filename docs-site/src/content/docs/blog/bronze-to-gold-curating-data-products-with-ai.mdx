Every data team knows the feeling: you have a database with hundreds of tables, most of them undocumented, and the idea of cataloging everything feels impossible. So you do nothing. Or you start a documentation sprint that loses momentum after two weeks. Or you buy a data catalog tool that sits mostly empty because nobody has time to fill it in.

The problem is not laziness. The problem is that most approaches to data curation are all-or-nothing. Either you document everything to perfection or you have nothing useful. There is no middle ground.

ContextKit's tier system changes this. It defines three levels of metadata maturity — Bronze, Silver, and Gold — with clear criteria, incremental value, and AI-assisted tooling at every step. You start with what you can automate, improve what matters most, and reach production quality where it counts.

## The Three Tiers

### Bronze: Auto-Generated Schema Metadata

Bronze is the starting point, and it is free. Run a single command and ContextKit introspects your database, generating YAML metadata for every table and column:

```bash
context introspect
```

Bronze metadata includes:

- Table names and column names
- Data types and nullability
- Primary keys and foreign key relationships
- Basic structure of the schema

This is the same information an AI agent would get from querying `information_schema`, but structured as portable YAML files in your repository. It takes minutes to generate and costs zero human effort.

**What Bronze gives you:** AI agents now have a structured, organized view of your schema. They can look up table structures, understand data types, and follow relationships. This is already better than having no metadata at all — it prevents basic errors like querying nonexistent columns or using wrong data types.

**What Bronze lacks:** Business context. The agent knows there is a column called `amt` of type `decimal`, but it does not know that `amt` is transaction revenue in cents, should be aggregated with SUM, and requires a status filter.

### Silver: Human-Curated Business Context

Silver is where metadata becomes genuinely useful. It adds the business context that machines cannot infer from schemas alone:

- **Descriptions** for every table and column, written in plain English
- **Semantic roles** that classify columns as dimensions, measures, identifiers, or metadata
- **Tags** that organize columns into business domains (revenue, customer, product)
- **Relationship annotations** that clarify join semantics

Silver metadata for a column looks like:

```yaml
- name: amount
  type: decimal
  description: "Transaction total in USD cents, including tax, for completed purchases"
  semantic_role: measure
  aggregation: SUM
  tags: [revenue, financial, kpi]
```

**What Silver gives you:** AI agents can now write meaningfully correct queries. They know what columns mean, how to aggregate them, and how they relate to business concepts. An agent that reads Silver metadata will SUM the revenue column instead of averaging it, because the semantic role and aggregation are explicit.

**What Silver takes:** Human knowledge. Someone who understands the business needs to validate descriptions and assign semantic roles. But this is where AI dramatically accelerates the work.

### Gold: Production-Ready with Guardrails

Gold is the highest tier — metadata that is complete enough for an AI agent to answer common business questions correctly on the first try. Gold adds:

- **Guardrails** — required filters that prevent data errors:

```yaml
guardrails:
  - column: status
    operator: "="
    value: "completed"
    reason: "Exclude cancelled, refunded, and test transactions"
```

- **Golden queries** — verified SQL for common business questions:

```yaml
golden_queries:
  - question: "What is total revenue this month?"
    sql: |
      SELECT SUM(amount) / 100.0 AS revenue_usd
      FROM transactions
      WHERE status = 'completed'
        AND created_at >= date_trunc('month', current_date)
```

- **Complete documentation** — every table, column, and relationship is fully described with no gaps

**What Gold gives you:** Confidence. When a data product reaches Gold tier, you can trust that AI agents querying it will produce correct results for any question the metadata covers. Guardrails prevent the most common errors. Golden queries provide verified baselines. The metadata is comprehensive enough that agents rarely need to guess.

**What Gold takes:** Iteration. Gold is not achieved in a single session. It is the result of curating metadata over time, adding guardrails as you discover error patterns, and writing golden queries for the questions that matter most.

## AI-Assisted Curation

The tier system is designed to work with AI, not against it. At every level, AI does the heavy lifting while humans provide the knowledge that only humans have.

### Bronze: Fully automated

No human input needed. ContextKit connects to your database and generates Bronze metadata automatically. The AI agent runs `context introspect` and the files appear.

### Silver: AI proposes, human validates

Running `context enrich` in an AI coding assistant starts a conversational enrichment process. The agent reads your Bronze metadata and proposes:

- Descriptions based on column names, types, and patterns
- Semantic role assignments based on naming conventions
- Tags based on business domain inference

You review each proposal. The agent might suggest: *"The column `cust_ltv` appears to be customer lifetime value — a financial measure that should be aggregated with SUM. Does this sound right?"* You confirm, correct, or add nuance. The agent writes the YAML.

This is dramatically faster than writing metadata from scratch. A typical table with 15-20 columns can be enriched to Silver in 5-10 minutes of conversational review.

### Gold: Human-driven, AI-supported

Gold requires domain expertise that AI cannot generate on its own. You need to know which filters are required, what the correct SQL for a business question looks like, and what edge cases exist. But AI helps:

- The agent can propose guardrails based on patterns it observes in the data
- It can draft golden queries based on the Silver metadata and ask you to verify them
- It can run `context lint` to identify gaps and suggest what to add next

The AI is a tireless co-author. You are the domain expert who validates its work.

## The Practical Approach: Start Where It Matters

You do not need to reach Gold tier across your entire database. The practical approach:

1. **Introspect everything to Bronze.** This takes minutes and covers your whole schema. Every AI agent immediately gets a better starting point.

2. **Pick your top 3-5 data products.** These are the tables that get queried most, power your dashboards, or are most often the source of wrong answers. Enrich them to Silver.

3. **Push one data product to Gold.** Pick the single most critical data product — probably your revenue or customer table — and invest in guardrails and golden queries. This is your reference implementation.

4. **Expand coverage over time.** As teams encounter errors or add new data products, the tier system gives them a clear path: introspect to Bronze, enrich to Silver, harden to Gold.

This approach means you get value on day one (Bronze), meaningful improvement in the first week (Silver for key products), and production-grade confidence within a month (Gold for critical products).

## Getting Started

```bash
npx create-contextkit
context introspect       # Bronze in minutes
context enrich           # Silver with AI assistance
context build            # Check your tier status
context serve --stdio    # Serve to AI tools via MCP
```

Or tell your AI agent:

> *"Install @runcontext/cli and build a semantic layer for my database. Start with the transactions table and get it to Gold tier."*

Every data product starts at Bronze. Where it goes from there is up to you.
