Claude Code is a powerful AI coding assistant that can connect to databases, write SQL, and analyze data. But out of the box, it only sees your schema — table names, column names, and data types. It does not know your business rules, metric definitions, or the required filters that prevent wrong answers.

This tutorial walks you through building a semantic layer with ContextKit and serving it to Claude Code via Model Context Protocol (MCP). By the end, Claude Code will have access to your full data catalog — descriptions, semantic roles, guardrails, and golden queries — every time it writes a query against your data.

**Time required:** 15-30 minutes for a basic setup, depending on the size of your schema.

**Prerequisites:** Node.js 18+, a database you want to catalog (Postgres, MySQL, SQL Server, or SQLite), and Claude Code installed.

## Step 1: Install ContextKit

The fastest way to get started:

```bash
npx create-contextkit
```

This scaffolds a new ContextKit project with the default configuration. Alternatively, install the CLI globally:

```bash
npm install -g @runcontext/cli
```

Then initialize in your project directory:

```bash
context setup
```

You will be prompted to enter your database connection details. ContextKit supports Postgres, MySQL, SQL Server, and SQLite. Enter the connection string or individual parameters when prompted.

After setup, you will have a `.context/` directory in your project with a `context.config.yaml` file containing your configuration.

## Step 2: Introspect Your Database

Run the introspection command to pull your schema into ContextKit:

```bash
context introspect
```

This connects to your database, reads the schema, and generates YAML metadata files in the `.context/` directory. Each data product gets its own YAML file with table definitions, column types, and detected relationships.

The output is **Bronze tier** metadata — automatically generated, structurally complete, but without business context. Check what was generated:

```bash
context build
```

This validates your metadata and reports the current tier for each data product. You should see Bronze tier across the board.

## Step 3: Enrich with Business Context

This is where the semantic layer gets its value. Run the enrichment command:

```bash
context enrich
```

If you are running this inside Claude Code (which is recommended), the enrichment becomes a conversational process. Claude Code will:

1. Read the Bronze metadata for each table
2. Propose descriptions based on column names and types
3. Suggest semantic roles (dimension, measure, identifier, metadata)
4. Ask you to confirm, correct, or expand each suggestion

For example, Claude Code might propose:

> *"The column `txn_amt` in table `transactions` appears to be a transaction amount. I suggest the description 'Transaction amount in USD' with semantic_role: measure and aggregation: SUM. Does this look right?"*

You respond: *"Correct, but it is in cents, not dollars. Add that to the description."*

The agent updates the YAML accordingly. This conversational enrichment is dramatically faster than writing metadata by hand. A table with 15-20 columns typically takes 5-10 minutes.

After enriching your core tables, run the build command again:

```bash
context build
```

You should now see Silver tier for enriched data products.

## Step 4: Add Guardrails and Golden Queries (Optional but Recommended)

For your most important data products, add guardrails and golden queries to reach Gold tier.

**Guardrails** are required filters that prevent common data errors. Add them to your YAML files:

```yaml
guardrails:
  - column: status
    operator: "="
    value: "completed"
    reason: "All revenue queries must exclude cancelled and refunded orders"
  - column: is_test
    operator: "="
    value: false
    reason: "Exclude test transactions from all metrics"
```

**Golden queries** are verified SQL for common business questions:

```yaml
golden_queries:
  - question: "What is total revenue this month?"
    sql: |
      SELECT SUM(txn_amt) / 100.0 AS revenue_usd
      FROM transactions
      WHERE status = 'completed'
        AND is_test = false
        AND created_at >= date_trunc('month', current_date)
  - question: "How many active customers do we have?"
    sql: |
      SELECT COUNT(DISTINCT id) AS active_customers
      FROM customers
      WHERE status = 'active'
        AND is_test = false
```

You can write these by hand or ask Claude Code to draft them based on your Silver metadata and then verify them. Run the lint command to check for issues:

```bash
context lint
```

And build again to confirm Gold tier:

```bash
context build
```

## Step 5: Configure the MCP Server for Claude Code

Now, serve your semantic layer to Claude Code via MCP. The MCP server is built into ContextKit — you just need to configure Claude Code to connect to it.

Add the following to your Claude Code MCP configuration file. The location depends on your setup:

- **Project-level:** `.claude/mcp.json` in your project root
- **User-level:** `~/.claude/mcp.json`

```json
{
  "mcpServers": {
    "contextkit": {
      "command": "context",
      "args": ["serve", "--stdio"]
    }
  }
}
```

If you installed ContextKit locally (not globally), use `npx`:

```json
{
  "mcpServers": {
    "contextkit": {
      "command": "npx",
      "args": ["@runcontext/cli", "serve", "--stdio"]
    }
  }
}
```

Save the file and restart Claude Code. The MCP server will start automatically when Claude Code launches.

## Step 6: Verify the Connection

Open Claude Code and ask it a question about your data. You should see it access the ContextKit MCP server to look up metadata before writing SQL.

Try a test query:

> *"What is total revenue this month?"*

Claude Code should:

1. Query the MCP server for the relevant table metadata
2. Read the description, semantic role, and aggregation for the revenue column
3. Apply any guardrails (like filtering for completed transactions)
4. Reference the golden query if one exists
5. Write SQL that matches your business definitions

Compare the result with and without the MCP server. Without it, Claude Code guesses at aggregation, filters, and semantics. With it, the query should match your golden query or follow your guardrails.

## Step 7: Iterate and Expand

Your semantic layer is live. From here:

**Add more data products.** Run `context introspect` as you add new tables or databases. Each new data product starts at Bronze and can be enriched incrementally.

**Refine based on errors.** When Claude Code writes a wrong query, ask yourself: what metadata was missing? Add the guardrail, update the description, or write a golden query. The semantic layer improves with use.

**Run lint in CI.** Add `context lint` to your CI pipeline to catch metadata regressions — missing descriptions, invalid semantic roles, or incomplete guardrails.

**Share with your team.** The `.context/` directory is part of your git repository. When teammates pull the latest code, they get the latest metadata. When they start Claude Code, they get the same semantic layer you do.

## What the MCP Server Exposes

For reference, the ContextKit MCP server exposes the following to Claude Code:

- **Resources:** Table metadata, column metadata, relationships, guardrails, golden queries, and tier status for each data product
- **Tools:** Search across data products, look up specific tables or columns, query guardrails for a given table
- **Prompts:** Guided workflows for common tasks like writing a query or exploring a data product

Claude Code uses these automatically when answering data questions. You do not need to prompt it to use the MCP server — it discovers the available resources and tools on its own.

## Troubleshooting

**Claude Code does not show MCP resources.** Verify the configuration file path and restart Claude Code. Check that `context serve --stdio` runs successfully from the command line.

**Metadata is stale.** If you have updated your database schema, run `context introspect` again to refresh the Bronze metadata, then re-enrich as needed.

**Golden queries do not match.** Verify them against your database directly. Golden queries should be tested SQL that returns correct results.

## Getting Started

The complete sequence:

```bash
npx create-contextkit          # Install
context setup                  # Configure database connection
context introspect             # Generate Bronze metadata
context enrich                 # Enrich to Silver (with AI)
context build                  # Validate and check tiers
context serve --stdio          # Serve to Claude Code via MCP
```

Or simply tell Claude Code:

> *"Install @runcontext/cli and build a semantic layer for my database, then configure MCP so you can use it."*

From that point on, every data question you ask Claude Code is answered with the full context of your semantic layer behind it.
