title: context blueprint
description: Export portable data product YAML blueprints following the OSI specification.

The `blueprint` command exports your compiled context as portable **data product YAML** files that follow the [Open Semantic Interchange (OSI)](/concepts/osi/) specification. Each blueprint is a self-contained file containing the full schema, governance, business rules, metrics, relationships, and AI context for a model.

## Usage

```bash
context build              # compile context files first
context blueprint          # export all models as YAML blueprints
context blueprint orders   # export a single model
```

## Options

| Option | Description | Default |
|---|---|---|
| `[model]` | Model name to export (omit for all) | all models |
| `--context-dir <path>` | Path to context files | `./context` |
| `--out <path>` | Output directory for blueprint files | `./blueprints` |
| `--stdout` | Print to stdout instead of writing files | — |

## Examples

### Export all models

```bash
context blueprint
```

Writes one `*.data-product.osi.yaml` file per model to `./blueprints/`.

### Export a single model to stdout

```bash
context blueprint orders --stdout
```

### Custom output directory

```bash
context blueprint --out docs/data-products
```

## What it generates

Each blueprint YAML contains:

- **Header** — OSI version, model name, description
- **Governance** — owner, trust status, tier, tags
- **Glossary** — business terms linked to this model
- **Datasets** — tables, fields with semantic roles, aggregation rules, sample values
- **Relationships** — joins with cardinality
- **Metrics** — named calculations with AI context
- **Business rules** — enforcement constraints
- **Guardrails** — required filters for correct queries
- **Golden queries** — verified SQL examples

## Example output

```yaml
# orders.data-product.osi.yaml
# Portable data product blueprint — Open Semantic Interchange (OSI) v1.0
# Generated by ContextKit (context blueprint)

osi_version: "1.0"

semantic_model:
  name: orders
  description: >
    Order transactions from the e-commerce platform
  owner: data-team
  trust_status: verified
  tier: gold
  tags:
    - revenue
    - e-commerce

  datasets:
    - name: orders
      source: analytics.fct_orders
      grain: order_id
      fields:
        - name: revenue
          expression: order_total
          description: Total order amount in USD
          semantic_role: measure
          aggregation: sum
          ai_context:
            instructions: >
              Always aggregate with SUM. Never AVG directly.
            synonyms:
              - sales
              - order amount

  relationships:
    - name: orders_to_customers
      from:
        dataset: orders
        columns: [customer_id]
      to:
        dataset: customers
        columns: [customer_id]
      cardinality: many_to_one

  metrics:
    - name: total_revenue
      expression: SUM(revenue)
      description: Sum of all order revenue
```

## Portability

Blueprint YAMLs are designed to be portable:

```bash
# Move a data product to another project
cp blueprints/orders.data-product.osi.yaml /other-project/context/

# Share with another team
context blueprint --out /shared/data-products/

# Serve via MCP — agents can also request blueprints dynamically
context serve --stdio
# Agent reads: context://data-product/orders
```
