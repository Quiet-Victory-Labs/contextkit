## Tell Your Agent to Build Your Semantic Layer

ContextKit is a CLI that AI agents use to build and serve semantic layers for your databases. Point an agent at your data, and it runs ContextKit commands to introspect the schema, scaffold metadata, enrich it with descriptions and types, and work with you to curate it to Gold tier — where agents can generate correct SQL autonomously.

```
You: "Build my semantic layer for this DuckDB"
Agent: runs context setup → introspects → scaffolds metadata
Agent: runs context enrich → auto-fills descriptions, types, lineage
Agent: runs context tier → "You're at Silver. Gold needs semantic roles and golden queries."
You: "The revenue field should always be SUM'd, never AVG'd"
Agent: updates governance YAML → runs context lint → "3 issues remaining"
Agent: runs context tier → "Gold ✓"
Agent: runs context serve → MCP server live, all agents get curated context
```

Built on the **[Open Semantic Interchange (OSI)](https://github.com/open-semantic-interchange/OSI)** specification — an open standard for describing semantic models in a vendor-neutral format. Your metadata is portable across any OSI-compliant tool, not locked into ContextKit.

## The Problem

AI agents querying databases see table and column names — but they don't understand what the data *means*. They don't know which filters are required, how metrics should be aggregated, or which joins are safe. Every query is a guess.

## The Solution

ContextKit creates a governance layer between your database and your AI tools:

```
Database  →  ContextKit  →  AI Agent
(schema)     (meaning)      (correct SQL)
```

With ContextKit, an agent knows:
- That `revenue` should be `SUM`'d, never `AVG`'d
- That queries on `transactions` must filter `WHERE status = 'completed'`
- That `customer_id` joins `orders` to `customers` as many-to-one
- That "demand signal" means review keywords like *wait*, *crowded*, *busy*
- Exactly which SQL to use for "show me top customers by revenue last quarter"

## How Agents Use It

### 1. As an MCP server (recommended)

Add ContextKit to your agent's MCP config:

```json
{
  "mcpServers": {
    "contextkit": {
      "command": "npx",
      "args": ["@runcontext/cli", "serve", "--stdio"]
    }
  }
}
```

The agent gets access to `context_search`, `context_explain`, `context_validate`, `context_tier`, `context_golden_queries`, and `context_guardrails` — everything needed to build and verify the metadata interactively.

### 2. As CLI commands

Any agentic coding platform (Claude Code, Cursor, Windsurf) can run ContextKit's 15 commands directly:

```bash
context setup          # interactive wizard
context introspect     # scaffold from database
context enrich         # auto-fill metadata
context lint           # check what's missing
context tier           # see maturity score
context serve          # serve to other agents
```

### 3. As a visual editor

For humans who want to curate without touching YAML:

```bash
context dev --studio
```

Opens an interactive browser UI where you edit descriptions, semantic roles, golden queries, and business rules. Changes save to YAML on disk and the tier scorecard updates live.
