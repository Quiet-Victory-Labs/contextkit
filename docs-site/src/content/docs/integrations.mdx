ContextKit works with any MCP-compatible AI tool. This page covers how to connect the most popular ones.

There are two ways to serve your semantic plane to AI agents:

- **BYOMCP (local):** Run the MCP server yourself via `context serve --stdio` or `context serve --http`. Free, works offline, your data stays on your machine.
- **Hosted MCP:** Connect to the managed endpoint at `mcp.runcontext.dev`. No server to run, shared team access, requires a [Team or Enterprise plan](/pricing/).

---

## Claude Code

Claude Code reads MCP configuration from `.claude/mcp.json` in your project root.

<Tabs>
<TabItem label="Local (stdio)">
```json title=".claude/mcp.json"
{
  "mcpServers": {
    "contextkit": {
      "command": "npx",
      "args": ["@runcontext/cli", "serve", "--stdio"]
    }
  }
}
```
</TabItem>
<TabItem label="Hosted">
```json title=".claude/mcp.json"
{
  "mcpServers": {
    "contextkit": {
      "url": "https://mcp.runcontext.dev/{org}",
      "headers": {
        "Authorization": "Bearer ${CONTEXTKIT_API_KEY}"
      }
    }
  }
}
```
</TabItem>
</Tabs>

Once configured, Claude Code discovers your semantic layer automatically. Try asking:

> *"Search my semantic layer for revenue metrics"*

## BYOMCP vs Hosted MCP

| | BYOMCP (Local) | Hosted MCP |
|---|---|---|
| **How it works** | You run `context serve` on your machine | ContextKit runs the MCP server at `mcp.runcontext.dev` |
| **Cost** | Free | [Team or Enterprise plan](/pricing/) |
| **Setup** | Install CLI, run `context serve --stdio` | Set API key, point client at hosted URL |
| **Team access** | Single user (unless you self-host HTTP) | Shared endpoint, all team members connect |
| **Data location** | Your machine / your infrastructure | Hosted by ContextKit |
| **Offline support** | Yes | No |
| **Connectors** | Connect from your machine | Managed connectors (3 on Team, unlimited on Enterprise) |
| **RBAC** | No | Enterprise plan |

### When to use BYOMCP

- You're a solo developer
- You want your data to stay on your machine
- You're working offline or in an air-gapped environment
- You don't need shared team access

### When to use Hosted MCP

- Your team needs shared access to the same semantic plane
- You don't want to run infrastructure
- You need RBAC or SSO
- You want managed database connectors

## MCP Tools Available

Once connected (via either method), your AI agent can call these MCP tools:

| Tool | Description |
|------|-------------|
| `context_search` | Search the semantic layer by keyword or concept |
| `context_explain` | Get the full context for a dataset or field — descriptions, rules, guardrails |
| `context_validate` | Validate a SQL query against guardrails and semantic rules |
| `context_tier` | Check the maturity tier (Bronze/Silver/Gold) of a data product |
| `context_golden_queries` | Retrieve pre-approved SQL queries for common business questions |
| `context_guardrails` | Get required filters, forbidden joins, and other safety rules |

For full MCP server documentation, see the [MCP Server reference](/mcp/overview/).

## Verifying Your Connection

After configuring your client, verify the connection by asking your AI agent:

> *"What MCP resources are available?"*

It should list the `context://` resources. Then test a tool call:

> *"Search my semantic layer for revenue metrics."*

The agent should call `context_search` and return matching datasets and fields from your semantic plane.
