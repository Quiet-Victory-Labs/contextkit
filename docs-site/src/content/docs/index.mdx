## The Problem

AI agents can see your table and column names. They can't see what the data **means**. They don't know which filters are required, how metrics aggregate, or which joins are safe.

<div class="comparison">
  <div class="without">
    <h4>Without ContextKit</h4>
    <pre><code>-- "What's total revenue?"
SELECT AVG(revenue)     ← wrong aggregation
FROM transactions       ← no status filter
                        ← includes cancelled</code></pre>
  </div>
  <div class="with">
    <h4>With ContextKit</h4>
    <pre><code>-- "What's total revenue?"
SELECT SUM(revenue)     ← correct (semantic role)
FROM transactions
WHERE status='completed' ← required (guardrail)</code></pre>
  </div>
</div>

## Three Tiers to AI-Ready

<CardGrid>
  <Card title="Bronze — Discoverable" icon="document">
    Descriptions, ownership, security, grain statements. Achieved automatically via `context introspect`.
  </Card>
  <Card title="Silver — Trusted" icon="approve-check">
    Trust status, glossary links, lineage, sample values. Achieved via `context enrich --target silver`.
  </Card>
  <Card title="Gold — AI-Ready" icon="star">
    Semantic roles, aggregation rules, guardrails, golden queries, business rules. Requires human curation.
  </Card>
  <Card title="40 Lint Rules" icon="list-format">
    Schema validation, governance, security, data accuracy — all tier-aware with auto-fix support.
  </Card>
</CardGrid>

## Quick Start

```bash
npm install -g @runcontext/cli
context setup           # interactive wizard — database to metadata
context tier            # check your Bronze/Silver/Gold score
context serve           # expose to AI agents via MCP
```
