## The Problem

You asked your AI agent: *"What was total revenue last quarter?"*

It connected to your database. It wrote SQL. It ran. It returned a number.

**That number was wrong.**

The agent used `AVG` instead of `SUM`. It forgot to filter out cancelled orders. It joined to the wrong table. And it did all of this confidently, without telling you anything was off.

This happens because AI agents can see your schema — table names, column names, types — but they can't see what the data *means*. They don't know which aggregation to use, which filters are required, which joins are safe, or what "revenue" actually measures in your business.

## The Fix: A Semantic Plane

A **semantic plane** is a structured metadata layer that sits between your database and your AI tools. It captures what your data means — field descriptions, business rules, metric definitions, safe joins, guardrail filters — so agents generate correct queries instead of guessing.

```
Your Database  →  Semantic Plane  →  AI Agent
(raw schema)      (meaning)          (correct SQL)
```

ContextKit builds this layer for you. You describe your data once, and every AI tool in your stack gets the same understanding.

## How It Works

### 1. Fill Out a Context Brief

Run `context setup` and a wizard opens in your browser. You'll answer five simple questions:

- **What is this data product?** — Give it a name and description
- **Who owns it?** — Name, team, email
- **How sensitive is it?** — Public, internal, confidential, or restricted
- **Where does the data live?** — Connect your database (auto-detected from your environment)
- **Any existing docs?** — Upload SQL files, data dictionaries, markdown — anything that helps

This takes about two minutes. You're describing what you already know about your data.

### 2. The Pipeline Builds Your Semantic Plane

Once you submit the Context Brief, ContextKit runs an automatic pipeline:

- **Introspect** — connects to your database, reads every table, column, and type
- **Scaffold** — generates Bronze-tier metadata (descriptions, ownership, security classification)
- **Enrich** — auto-fills Silver-tier context (glossary terms, lineage, sample values, trust status)
- **Verify** — lints everything and flags issues

You watch each stage complete in the browser. When it's done, you have a Silver-tier semantic plane — already useful, already serving context to AI agents.

### 3. Curate to Gold

Silver gets you most of the way. Gold is what makes agents *truly* reliable.

Open the visual studio (`context dev --studio`) and add the business knowledge that only you have:

- **Semantic roles** — "this column is a metric, always SUM it"
- **Guardrail filters** — "always filter WHERE status = 'completed'"
- **Golden queries** — "the correct SQL for 'total revenue' is..."
- **Business rules** — "revenue = price * quantity, excluding refunds"

This is the work that prevents wrong answers. It takes time, but it's the most valuable thing you can do for your data.

### 4. Serve to Every AI Tool

```bash
context serve --stdio
```

Your semantic plane is now available to every MCP-compatible AI tool — Claude Code, Cursor, Copilot, Windsurf, Codex. Agents can search your metadata, look up business terms, check guardrails, and find golden queries.

One source of truth. Every tool gets it.
