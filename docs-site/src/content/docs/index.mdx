## The Semantic Plane for Your Data

AI agents see table and column names, but they don't understand what the data **means**. They don't know which filters are required, how metrics should be aggregated, or which joins are safe. Every query is a guess.

ContextKit creates a **semantic plane** — a structured layer of metadata that sits between your database and your AI tools. It tells agents the meaning behind the schema so they generate correct SQL on the first try.

```
Database  →  Semantic Plane  →  AI Agent
(schema)     (meaning)          (correct SQL)
```

## How It Works

You tell your AI agent to build it. The agent runs CLI commands, asks you about your data, and iterates until Gold. You provide the business knowledge — the agent does the rest.

```
You: "Install @runcontext/cli and build a semantic layer for my database"
Agent: npm install @runcontext/cli
Agent: context setup → connects → introspects → scaffolds metadata
Agent: context enrich --target silver --apply → auto-fills descriptions, types, lineage
Agent: context tier → "Silver. 12 Gold checks remaining."
Agent: "I see a revenue column — should this be SUM'd or AVG'd?"
You: "Always SUM. And filter out refunded orders."
Agent: updates governance YAML, adds guardrail filter
Agent: context tier → ... iterates until Gold
Agent: context serve --stdio → MCP server live
```

## Architecture

ContextKit runs **local-first** — your metadata lives in your repo as YAML files. No cloud account needed, no vendor lock-in. When you're ready for teams, connect the hosted semantic plane.

```
┌─────────────────────────────────────────────────┐
│  Your Repository                                │
│  context/                                       │
│    products/                                    │
│      my_product/                                │
│        osi.yaml          ← OSI spec (portable)  │
│        governance.yaml   ← rules & guardrails   │
│        owner.yaml        ← ownership            │
│    contextkit.config.yaml                       │
└───────────────┬─────────────────────────────────┘
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  CLI        MCP Server   Static Site
 (build,     (serve to    (browsable
  lint,       agents)      docs)
  enrich)
```

**Local-first.** Everything runs on your machine. No API keys, no cloud dependency.

**Cloud-ready.** When your team needs shared access, connect the hosted semantic plane at `mcp.runcontext.dev` for centralized serving with RBAC and connectors.

## Quick Start

```bash
npx create-contextkit
# or manually:
npm install -g @runcontext/cli
context setup           # connect → introspect → scaffold → enrich
context tier            # check your Bronze/Silver/Gold score
context serve --stdio   # serve to AI agents via MCP
```

Or just tell your AI agent:

> *"Install @runcontext/cli and build a semantic layer for my database."*
