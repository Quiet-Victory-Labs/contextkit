## From Database to AI-Ready in Four Steps

AI agents see table and column names, but they don't understand what the data **means**. They don't know which filters are required, how metrics should be aggregated, or which joins are safe. Every query is a guess.

ContextKit fixes this by building a **semantic plane** — a structured metadata layer between your database and your AI tools.

```
1. context setup          → Browser wizard opens
2. Fill out Context Brief → Name, owner, sensitivity, database
3. Pipeline auto-runs     → Introspect → Scaffold → Enrich → Verify
4. context serve          → MCP server live — agents have full context
```

## How It Works

**Step 1 — Context Brief.** Run `context setup` and a 5-step wizard opens in your browser. Name your data product, set an owner, choose a sensitivity level (public, internal, confidential, restricted), and connect your database. Upload any existing documentation — SQL files, data dictionaries, markdown — to give the enrichment engine more to work with.

**Step 2 — Automatic Pipeline.** ContextKit introspects your schema, scaffolds Bronze-tier metadata (descriptions, types, ownership), then auto-enriches toward Silver (trust status, glossary, lineage, sample values). The pipeline runs in real time — you watch each stage complete in the browser.

**Step 3 — Curate to Gold.** Open the visual studio (`context dev --studio`) to add semantic roles, aggregation rules, guardrail filters, golden queries, and business rules. Gold tier requires your business knowledge — and that's where the real value is.

**Step 4 — Serve via MCP.** Run `context serve` and every AI tool in your stack gets access to your semantic plane. Agents can search metadata, look up business terms, retrieve guardrails, and find golden queries — all through the Model Context Protocol.

Or skip the wizard and tell your AI agent:

> *"Install @runcontext/cli and build a semantic layer for my database."*

The agent runs the same pipeline programmatically, asking you about metrics, ownership, and business rules as it goes.

## Architecture

ContextKit runs **local-first** — your metadata lives in your repo as YAML files. No cloud account needed, no vendor lock-in.

```
┌─────────────────────────────────────────────────┐
│  Your Repository                                │
│  context/                                       │
│    products/                                    │
│      my_product/                                │
│        context-brief.yaml  ← your data product  │
│        osi.yaml            ← OSI spec (portable) │
│        governance.yaml     ← rules & guardrails  │
│    contextkit.config.yaml                       │
└───────────────┬─────────────────────────────────┘
                │
    ┌───────────┼───────────────┐
    ▼           ▼               ▼
  Setup      MCP Server      Visual Studio
  Wizard     (serve to        (curate to
  (build)     agents)          Gold tier)
```

**Local-first.** Everything runs on your machine. No API keys, no cloud dependency.

**Cloud-ready.** When your team needs shared access, **[RunContext Cloud](https://runcontext.dev)** adds multi-user curation, hosted MCP serving, and managed infrastructure on top of the same open core.

## Quick Start

```bash
npx create-contextkit my-data
cd my-data
context setup           # browser wizard — database to data product
context tier            # check your Bronze/Silver/Gold score
context serve --stdio   # serve to AI agents via MCP
```
